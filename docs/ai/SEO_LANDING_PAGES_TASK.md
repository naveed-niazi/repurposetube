This file defines the SEO landing page workstream for AI and developers.
Goal: create high-quality, indexable landing pages that target MVP features and rank for relevant intent.

## SEO Positioning (Core Strategy)

**Own this angle:** *YouTube URL → 6 written content formats in parallel, in 60 seconds.*

Most competitors are either podcast-first (Castmagic), video-clip-first (Opus Clip, Vidyo), or single-format
(VideoToBlog.ai). This product is unique because it is:

- **YouTube-native** — URL input, not file upload
- **Writing/text-first** — outputs are blog, thread, newsletter, not more video
- **Complete set** — all 6 formats in one click, parallel generation
- **YouTube SEO-inclusive** — A/B titles + full description + 15 tags bundled into the repurposing workflow

No competitor bundles all 6 written output formats into a single YouTube URL workflow. That is the moat.

The "Castmagic alternative for YouTubers" framing is highly rankable and accurate. Castmagic is podcast-first;
this product is YouTube-first. Comparison pages targeting "[competitor] alternative" keywords convert fast and
rank faster than category pages for a new site.

## Objective

- Build multiple SEO landing pages, one per core feature/theme.
- Reuse shared page components to keep implementation consistent and fast.
- Maximize technical and on-page SEO quality for strong ranking potential.

## Scope

- Create dedicated landing pages for feature-focused keywords.
- Use shared section components (hero, benefit blocks, FAQ, CTA, social proof pattern).
- Add complete SEO metadata and structured data to every page.
- Ensure pages are crawlable, fast, and internally linked.

## Non-Negotiable Rules

- Reuse existing shared UI blocks; avoid one-off page implementations unless necessary.
- Each page must have unique title, description, H1, and primary keyword intent.
- Do not duplicate the same copy across pages.
- Keep content user-helpful first; avoid keyword stuffing.

---

## SEO Implementation Checklist (Per Page)

- [ ] Unique SEO title (target keyword near start, natural wording).
- [ ] Unique meta description (clear value proposition + CTA intent).
- [ ] Canonical URL configured.
- [ ] Open Graph tags (title, description, URL, image).
- [ ] Twitter/X card tags.
- [ ] Proper heading hierarchy (`H1` once, logical `H2/H3` sections).
- [ ] Descriptive internal links to relevant product pages.
- [ ] FAQ section with `FAQPage` JSON-LD when applicable.
- [ ] Breadcrumb JSON-LD when page is in a hierarchy.
- [ ] Valid index/follow settings (no accidental noindex on public pages).
- [ ] SEO-friendly slug and clean URL path.
- [ ] Optimized images with descriptive alt text.

## Technical SEO Checklist (Site-Wide)

- [ ] XML sitemap includes all landing pages.
- [ ] `robots.txt` allows crawling of public landing pages.
- [ ] No broken internal links.
- [ ] No duplicate canonical conflicts.
- [ ] Good Core Web Vitals baseline for landing pages.
- [ ] Mobile-friendly layout and readable typography.

## Content Strategy Tasks

- [ ] Define target keyword for each feature page.
- [ ] Map one primary intent + 2-5 secondary terms per page.
- [ ] Add feature-specific examples/use cases.
- [ ] Include clear CTA sections tied to product action.
- [ ] Add concise FAQ answering intent-specific objections.

---

## Definition of Done

A landing page is done only when:

- All SEO implementation checklist items are complete.
- Content is unique and aligned to one feature intent.
- Shared components are reused (no unnecessary custom duplication).
- Page passes internal review for readability and conversion clarity.
- Lighthouse SEO score target is high (aim for 95+).
